---
title: 'CLDDownloader'
description: 'Asynchronously fetch images and raw assets from Cloudinary URLs, with built-in caching and progress tracking.'
---

`CLDDownloader` fetches remote resources from Cloudinary. It serves images from the on-device URL cache when available, and downloads and caches them on a miss. Obtain an instance by calling `createDownloader()` on your `CLDCloudinary` object.

```swift
let downloader = cloudinary.createDownloader()
```

<Note>
  Do not instantiate `CLDDownloader` directly. Always obtain it through `CLDCloudinary.createDownloader()`.
</Note>

---

## Methods

### fetchImage(_:_:completionHandler:)

Asynchronously fetches an image from the given URL. The image is returned from the cache when present; otherwise it is downloaded, cached, and returned.

```swift
@discardableResult
open func fetchImage(
    _ url: String,
    _ progress: ((Progress) -> Void)? = nil,
    completionHandler: CLDCompletionHandler? = nil
) -> CLDFetchImageRequest
```

<ParamField path="url" type="String" required>
  The full URL string of the image to fetch. Use a URL generated by `CLDUrl` to ensure the correct transformation parameters are included.
</ParamField>

<ParamField path="progress" type="((Progress) -> Void)?">
  A closure called periodically during the download. The `Progress` object exposes `completedUnitCount` and `totalUnitCount` in bytes.
</ParamField>

<ParamField path="completionHandler" type="CLDCompletionHandler?">
  Called once the fetch completes. The signature is `(UIImage?, NSError?) -> ()`. Exactly one of the arguments is non-nil.
</ParamField>

<ResponseField name="return" type="CLDFetchImageRequest">
  A request handle conforming to `CLDFetchImageRequest`. Use it to register additional response closures or to cancel, suspend, or resume the download.
</ResponseField>

```swift
let imageUrl = cloudinary.createUrl().generate("sample") ?? ""

downloader.fetchImage(imageUrl) { image, error in
    DispatchQueue.main.async {
        imageView.image = image
    }
}
```

### fetchAsset(_:_:completionHandler:)

Asynchronously fetches a raw asset from the given URL and returns it as `Data`. Useful for non-image resources such as video, PDF, or raw files where you need the binary content directly.

```swift
@discardableResult
open func fetchAsset(
    _ url: String,
    _ progress: ((Progress) -> Void)? = nil,
    completionHandler: CLDAssetCompletionHandler? = nil
) -> CLDFetchAssetRequest
```

<ParamField path="url" type="String" required>
  The full URL string of the asset to fetch.
</ParamField>

<ParamField path="progress" type="((Progress) -> Void)?">
  A closure called periodically during the download.
</ParamField>

<ParamField path="completionHandler" type="CLDAssetCompletionHandler?">
  Called once the fetch completes. The signature is `(Data?, NSError?) -> ()`. Exactly one of the arguments is non-nil.
</ParamField>

<ResponseField name="return" type="CLDFetchAssetRequest">
  A request handle conforming to `CLDFetchAssetRequest`. Use it to register additional response closures or to cancel, suspend, or resume the download.
</ResponseField>

```swift
downloader.fetchAsset(pdfUrl) { data, error in
    if let data = data {
        try? data.write(to: localFileUrl)
    }
}
```

## CLDFetchImageRequest

`CLDFetchImageRequest` is the handle returned by `fetchImage`. It extends `CLDNetworkDataRequest` with an image-specific response method.

### responseImage(_:)

```swift
@discardableResult
func responseImage(
    _ completionHandler: CLDCompletionHandler?
) -> CLDFetchImageRequest
```

Registers a completion closure that receives the fetched `UIImage`. You can call this multiple times to add multiple observers; each closure is called once the image is available.

<ParamField path="completionHandler" type="CLDCompletionHandler?">
  `(UIImage?, NSError?) -> ()` â€” Called with the fetched image or an error.
</ParamField>

<ResponseField name="return" type="CLDFetchImageRequest">
  The same request instance, enabling chaining.
</ResponseField>

```swift
downloader
    .fetchImage(url)
    .responseImage { image, error in
        imageView.image = image
    }
```

## Caching

`CLDDownloader` uses a `URLCache` shared through `CLDDownloadCoordinator`. You can control caching behavior through the `CLDCloudinary` properties:

| Property | Default | Description |
|---|---|---|
| `cloudinary.enableUrlCache` | `true` | Enable or disable the cache entirely. |
| `cloudinary.cacheMaxDiskCapacity` | 150 MB | Maximum on-disk cache size in bytes. |
| `cloudinary.cacheMaxMemoryTotalCost` | 30 MB | Maximum in-memory cache size in bytes. |

```swift
cloudinary.enableUrlCache = true
cloudinary.cacheMaxDiskCapacity = 200 * 1024 * 1024   // 200 MB
cloudinary.cacheMaxMemoryTotalCost = 50 * 1024 * 1024  // 50 MB
```

<Tip>
  For background download sessions, register a completion handler in your app delegate using `CLDCloudinary.setBackgroundCompletionHandler(_:)` so the system can notify your app when background transfers finish.
</Tip>
